# Webness User Flow

Date: June 4, 2026

## Main Flow

1. User lands on homepage.
2. User sees Webness positioning and proof.
3. User reviews work or service lanes.
4. User learns Signal Room diagnoses before Webness builds.
5. User applies for Signal Scan.
6. Webness reviews fit.
7. Webness offers paid Signal Scan or declines.
8. Signal Scan produces priority map.
9. Priority map routes into sprint or retainer.
10. Signal Room tracks next improvements after launch.

## Application Fields

- Name.
- Business.
- Website or product URL.
- Business type.
- Email.
- Phone or WhatsApp.
- Target market.
- Budget range.
- Problem to diagnose.
- Desired outcome.
- Timeline.

## Success Criteria

The user should never be unsure whether Webness sells websites/apps/services. Signal Room should feel like the decision engine behind those services, not a replacement for them.
