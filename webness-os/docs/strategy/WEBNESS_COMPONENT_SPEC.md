# Webness Component Spec

Date: June 4, 2026

## Components Implemented

- `PremiumHero`: first-viewport positioning and CTA.
- `PortfolioProofGrid`: all case-study proof categories.
- `ServiceLanes`: what Webness builds and top service lanes.
- `SignalRoomPreview`: diagnostic cards and funnel explanation.
- `Webness100Scorecard`: internal quality gate.
- `OfferLadder`: paid scan, builds, and retainers.
- `ProcessTimeline`: diagnose -> design -> build -> launch -> track -> improve.
- `FounderProof`: founder-led trust and best-fit clients.
- `CaseStudyCard`: reusable work proof card.
- `ApplyForm`: application capture and validation.
- `FAQ`: public objections.
- `FinalCTA`: final application CTA.

## Design Rules

- Keep the UI premium and restrained.
- Use proof before promises.
- Avoid decorative clutter.
- Keep CTAs clear and repeated.
- Avoid the phrase "AI agency" as primary positioning.
